# Claude Code Guidelines for CheapQuickVegan

This document contains important guidelines and context for working on this project.

## Project Overview

CheapQuickVegan is a Next.js-based vegan food and travel blog with a focus on SEO optimization and user experience.

## Critical SEO Rules

### NEVER Change URL Structures

**CRITICAL:** Do not modify URL patterns or add query parameters to existing routes. URL changes harm SEO by:
- Creating duplicate content issues
- Breaking external links and backlinks
- Losing accumulated search engine rankings
- Fragmenting page authority across multiple URLs

**Examples of prohibited changes:**
- ❌ Adding query parameters: `/recipes/recipe-slug?from=breakfast`
- ❌ Changing route patterns: `/recipes/[slug]` → `/r/[slug]`
- ❌ Adding optional parameters that create alternate URLs
- ❌ Modifying canonical URL structures

**Allowed changes:**
- ✅ Adding new routes that don't conflict with existing URLs
- ✅ Internal state management (client-side only, no URL changes)
- ✅ Fragment identifiers for in-page navigation (e.g., `#section`)

### URL Structure Rules

Current URL patterns that must remain unchanged:
- `/` - Homepage
- `/recipes` - All recipes listing
- `/recipes/[slug]` - Individual recipe pages
- `/recipes/category/[category]` - Category pages
- `/about` - About page
- `/favorites` - Favorites page

### Breadcrumb & Navigation Context

If you need to provide context about where a user came from:
- Use client-side state management (React state, context, localStorage)
- Use session storage for temporary navigation state
- Never reflect this in the URL

## Architecture Notes

- **Framework:** Next.js 16 with App Router
- **Content Source:** Notion API (cached)
- **Styling:** Tailwind CSS with shadcn/ui components
- **Database:** Vercel Postgres (for comments and ratings)

## SEO Priorities

1. Clean, semantic URL structures
2. Proper schema.org markup (Recipe, BreadcrumbList)
3. Canonical URLs in metadata
4. OpenGraph and Twitter card tags
5. Sitemap generation
6. Image optimization with alt text

### Web Manifest

**NEVER remove `manifest: "/site.webmanifest"` from `src/app/layout.tsx`**
- The web manifest is important for SEO and discoverability
- It helps search engines understand the site structure
- It enables the site to be installable as a PWA (Progressive Web App)
- Removing it can negatively impact search rankings

## Development Guidelines

- Always use the recipe's primary category (first in array) for breadcrumbs
- Maintain consistent internal linking patterns
- Ensure all recipe links point to `/recipes/[slug]` without query parameters
- Keep structured data (JSON-LD) synchronized with page content
